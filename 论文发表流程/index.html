    为什么先发 arXiv？¶

    1. 建立时间戳：arXiv 的提交时间是公开的，证明工作优先权
    2. 快速传播：学术论文引用 arXiv 预印本已成惯例
    3. 获取反馈：发表后可收集社区意见，改进正式投稿版本
    4. HypRAG/HyperbolicRAG 都先发了 arXiv：竞品策略验证

    详细流程¶

    Phase A: arXiv 发布（预计 1-2 周）¶

    Step A1: 论文准备¶

    • 确定论文标题（中英文版本）
    • 完成摘要（5句式公式：What/Why/How/Evidence/Remarkable）
    • 完成引言（≤1.5页，2-4条贡献）
    • 完成方法章节（含数学公式）
    • 完成实验章节（真实数据 + 消融实验）
    • 完成讨论和结论
    • 完成参考文献（25-40篇）
    • 制作 Figure 1（系统架构图，最关键的图）
    • 制作实验结果图表（表格 + 柱状图）

    Step A2: 格式检查¶

    arXiv 接受以下格式： - LaTeX（推荐） - PDF（直接上传）

    推荐使用 LaTeX：

    # 编译 LaTeX
cd paper/ieee/
xelatex ucef-en
bibtex ucef-en
xelatex ucef-en
xelatex ucef-en

    arXiv 要求： - [ ] 无版权保护的字体 - [ ] 所有图片嵌入或附源文件 - [ ] .bbl 文件内嵌（或使用 biblatex + .bcb） - [ ] 单文件提交或 zip 打包

    Step A3: 提交清单¶

    • 在 https://arxiv.org 注册账号
    • 填写元数据：标题、摘要、作者、分类
    • arXiv 分类选择：
    • 主分类：cs.CL (Computation and Language)
    • 辅分类：cs.AI, cs.IR
    • 上传 LaTeX 源文件（含图片）
    • 预览 PDF 检查排版
    • 点击 "Submit" 发布

    Step A4: 发布后¶

    • 在 GitHub README 中添加 arXiv 链接和 badge
    • 在 Twitter/X、Reddit r/MachineLearning 宣传
    • 在相关 GitHub issue/discussion 中引用
    • 通知 HypRAG/HyperbolicRAG 作者（建立学术联系）

    Phase B: 中文期刊投稿（计算机学报）¶

    Step B1: 格式适配¶

    计算机学报使用 LaTeX 模板：

    # 已有模板
cd paper/chinese-journal/
xelatex ucef-cn
bibtex ucef-cn
xelatex ucef-cn
xelatex ucef-cn

    中文期刊特殊要求： - [ ] 中文摘要 300-500 字 - [ ] 英文摘要对应翻译 - [ ] 中图分类号：TP391（信息处理） - [ ] 关键词 3-8 个（中英文对照） - [ ] 基金项目信息（如适用）

    Step B2: 论文结构调整¶

    中文期刊论文结构：

    摘要 → 关键词
  ↓
1 引言
2 相关工作
3 问题形式化
4 方法
  4.1 双曲几何检索
  4.2 量子启发选择
  4.3 自适应压缩
  4.4 质量保持引擎
  4.5 三层记忆架构
5 实验
  5.1 实验设置
  5.2 主要结果
  5.3 消融实验
  5.4 案例分析
6 讨论
7 结论与展望
参考文献（30-50篇）

    Step B3: 投稿¶

    • 注册计算机学报在线投稿系统
    • 上传 LaTeX 源文件 + PDF
    • 填写投稿信息
    • 等待初审（1-2周）
    • 外审（2-4个月）

    Step B4: 审稿回复模板¶

    尊敬的编辑老师和审稿专家：

感谢对本文的审阅。针对审稿意见，我们逐条回复如下：

意见1：[审稿人原文]
回复：[逐条详细回复]
修改位置：[章节.页码.行号]

意见2：[审稿人原文]
回复：...

    Phase C: 英文顶会投稿¶

    Step C1: 目标会议选择策略¶

    会议 截止日 特点 UCEF 匹配度
    ACL 2027 2027.01 NLP 首选 极高（检索+选择+压缩都是 NLP）
    EMNLP 2026 2026.06 NLP 第二选择 极高
    ICLR 2027 2026.09 偏理论/表示学习 高（双曲几何创新）
    NeurIPS 2027 2027.05 ML 顶会 中（需强化理论贡献）
    ICML 2027 2027.01 ML 顶会 中（同上）

    Step C2: 论文差异化定位¶

    投 ACL/EMNLP：定位为 "NLP 系统论文" - 强调端到端 LLM 上下文扩展 - 在 NLP benchmark（LongBench, NarrativeQA）上评估 - 贡献 = 框架 + 方法 + 开源

    投 ICLR：定位为 "表示学习论文" - 强调双曲几何表示的优势 - 消融实验证明几何归纳偏置的贡献 - 贡献 = 理论洞察 + 实验验证

    Step C3: 审稿标准应对¶

    顶会审稿标准（以 ACL 为例）：

    标准 权重 UCEF 应对
    原创性 25% 首个统一框架 + 量子选择（无竞品）
    技术质量 25% 完整数学推导 + 消融实验 + 统计检验
    清晰度 20% 遵循 writing-guide.md 7原则
    意义 20% 解决实际瓶颈（4K→1M+）
    可复现性 10% 开源代码 + 详细实验设置

    关键风险与应对¶

    风险 概率 应对策略
    实验数据不够强 中 增加更多 baseline，消融实验补充
    HypRAG 等竞品先发顶会 中 强调 UCEF 是统一框架而非纯检索方法
    审稿人质疑模拟实验 高 优先跑真实 benchmark（LongBench）
    量子方法被质疑"噱头" 中 消融实验量化贡献，强调密度矩阵的数学基础
    中文期刊格式问题 低 已有 LaTeX 模板，按模板调整

    论文质量检查清单¶

    提交前必检（基于 research-paper-writing skill）¶

    叙事性： - [ ] 能用一句话陈述核心贡献 - [ ] 引言中 What/Why/So What 三支柱清晰 - [ ] 每个实验支撑一个具体 claim

    结构： - [ ] 摘要遵循 5 句式公式 - [ ] 引言 ≤1.5 页 - [ ] 方法在第 2-3 页开始 - [ ] 2-4 条贡献 bullet - [ ] 局限性章节存在

    写作： - [ ] 术语全文一致 - [ ] 无泛泛开头（"近年来大语言模型..."） - [ ] 主动语态（"We show" 非 "It is shown"） - [ ] 所有图有自解释 caption

    技术： - [ ] 所有引用通过 API 验证 - [ ] 误差棒 + 统计检验方法说明 - [ ] 计算资源记录 - [ ] 代码/数据可用性声明
